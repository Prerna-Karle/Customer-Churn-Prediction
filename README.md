📉 Customer Churn Prediction

An end-to-end machine learning project that predicts whether a telecom customer is likely to churn (cancel their subscription), based on the Telco Customer Churn dataset. The project covers the full ML lifecycle — data cleaning, EDA, handling class imbalance, model comparison, hyperparameter tuning, and deployment as an interactive web app.


## 📌 Project Overview

Customer churn directly impacts a subscription business's revenue. This project builds a classification model that flags customers at high risk of churning, so a business can proactively target them with retention offers.

**Key steps:**
1. Data cleaning and preprocessing
2. Exploratory Data Analysis (EDA)
3. Handling class imbalance with SMOTE
4. Comparing Decision Tree, Random Forest, and XGBoost with 5-fold cross-validation
5. Hyperparameter tuning with `GridSearchCV` + Stratified K-Fold CV
6. Overfitting check (train vs. test accuracy)
7. Feature importance analysis
8. Model deployment as a Streamlit web app

## 📊 Dataset

- **Source:** [Telco Customer Churn (Kaggle)](https://www.kaggle.com/datasets/blastchar/telco-customer-churn)
- 7,043 customers, 21 features (demographics, account info, services subscribed)
- Target: `Churn` (Yes/No)

## ⚙️ Tech Stack

- **Language:** Python
- **Data handling:** pandas, NumPy
- **Visualization:** Matplotlib, Seaborn
- **Modeling:** scikit-learn, XGBoost, imbalanced-learn (SMOTE)
- **Deployment:** Streamlit

## 🧠 Model

| Model | Cross-Val Accuracy (default params) |
|---|---|
| Decision Tree | ~0.78 |
| **Random Forest** | **~0.84** |
| XGBoost | ~0.83 |

Random Forest was selected and further tuned with `GridSearchCV` (Stratified 5-Fold CV, optimized for F1-score) to handle the class imbalance appropriately.

## 🚀 How to Run Locally

### 1. Clone the repo
```bash
git clone https://github.com/<your-username>/customer-churn-prediction.git
cd customer-churn-prediction
```

### 2. Create a virtual environment and install dependencies
```bash
python -m venv venv
source venv/bin/activate      # on Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 3. Run the notebook (optional — regenerates the .pkl files)
Open `Customer_Churn_Prediction_using_ML.ipynb` in Jupyter/Colab and run all cells.
This produces `customer_churn_model.pkl` and `encoders.pkl`.

### 4. Launch the web app
```bash
streamlit run app.py
```
The app opens at `http://localhost:8501`.

## 🌐 Deployment (Streamlit Community Cloud — free)

1. Push this repo to GitHub (include `app.py`, `requirements.txt`, `customer_churn_model.pkl`, `encoders.pkl`).
2. Go to [share.streamlit.io](https://share.streamlit.io) and sign in with GitHub.
3. Click **New app**, select your repo/branch, and set the main file path to `app.py`.
4. Click **Deploy**. You'll get a public URL to share and put on your resume.

## 📈 Results & Insights

- Contract type, tenure, and monthly charges are the strongest predictors of churn.
- Month-to-month contract customers churn at a much higher rate than 1–2 year contract customers.
- SMOTE improved recall on the minority (churn) class significantly compared to training on the imbalanced data directly.
